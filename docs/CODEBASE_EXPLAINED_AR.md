# شرح هيكل الكود (AyniqAndroid)

هذا الملف يشرح مكونات المشروع الرئيسية وكيف ترتبط ببعض.

## 1) التطبيق Android
المجلد: `app/src/main/java/com/example/ayniqbooking`

- `MainActivity.kt`
  - واجهة التطبيق بالكامل (Compose UI).
  - يحتوي 3 تبويبات:
    - **الحسابات**: إعدادات عامة + إضافة/تعديل الحسابات.
    - **التشغيل**: اختيار التاريخ/الأوقات والتحكم في تشغيل البحث.
    - **المحجوزات**: عرض نتائج الحجوزات الناجحة.
  - تمت إضافة:
    - AppCheck/Proxy على مستوى عام.
    - AppCheck/Proxy على مستوى كل حساب (Override اختياري).
    - أزرار تشغيل/Pause/إيقاف بالألوان (أخضر/أزرق/أحمر) لفردي وجماعي.
    - إدخال التاريخ بـ 3 سلايدرز حتى في تاريخ التشغيل.

- `Models.kt`
  - نماذج البيانات الأساسية:
    - `AccountSession`, `BookingPayload`, `ProxyConfig`, `AppSettings`.
  - تمت إضافة:
    - `AccountNetworkOverride` لإعدادات الشبكة لكل حساب.
    - `AccountRunState` لحالة التشغيل (RUNNING/PAUSED/STOPPED).
    - `normalizeToTomorrowIfPast` لضبط أي تاريخ قديم تلقائيًا إلى الغد.
    - `mergeEffectiveSettings` لدمج إعدادات الحساب مع الإعدادات العامة.

- `AyniqApi.kt`
  - مسؤول عن الاتصال بـ API:
    - OTP
    - Login
    - إرسال الحجز
  - الآن يستخدم الإعدادات الفعالة لكل حساب (عام أو Override) قبل أي طلب.

- `BookingEngine.kt`
  - محرك التكرار والإرسال على الحسابات/التواريخ/الأوقات.
  - يدعم الآن حالات التحكم للحسابات أثناء التشغيل:
    - تشغيل
    - Pause
    - إيقاف

- `SessionStore.kt`
  - تخزين الحالة المحلية عبر DataStore.
  - تمت إضافة `updateSessions(...)` لتحديث الحسابات بشكل جماعي (مثل تعميم AppCheck/Proxy).

- `GateApi.kt`
  - بوابة التحقق من سماح المستخدم بالدخول عبر `control_server`.

## 2) سيرفر التحكم
المجلد: `control_server`

- `main.py`
  - FastAPI server لإدارة المستخدمين وصلاحياتهم.
  - يحوي Dashboard ويب لإدارة حسابات الدخول (Admin).

## 3) سكربت بايثون
الملف: `fixed_script.py`

- نسخة سكربت تشغيل/مرجع.
- تم تحديثه لي:
  - يعدل التاريخ تلقائيًا للغد إذا المدخل قديم/غير صالح.
  - يشغل على كل الحسابات المحفوظة بدل حساب واحد.

## 4) تدفق العمل الكامل
1. تسجيل الدخول عبر بوابة السيرفر.
2. إعداد AppCheck/Proxy العام أو تخصيصه لكل حساب.
3. إضافة حسابات (OTP + بيانات المواطن + الدائرة).
4. من تبويب التشغيل:
   - اختيار تاريخ (3 سلايدرز).
   - اختيار باقة/أوقات.
   - تشغيل/ Pause / إيقاف فردي أو جماعي.
5. متابعة النتائج في تبويب المحجوزات.
